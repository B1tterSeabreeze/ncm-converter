# 🎵 NCM 音乐转换工具

> **📋 声明**：本工具基于 [taurusxin/ncmdump](https://github.com/taurusxin/ncmdump) 项目制作，核心解密引擎由 [taurusxin](https://github.com/taurusxin) 开发。感谢原作者的杰出工作！

> 网易云音乐 NCM 文件批量解密转换工具，基于 [ncmdump](https://github.com/taurusxin/ncmdump) v1.5.1

将网易云音乐下载的 `.ncm` 加密缓存文件批量解密为原始的 `MP3` 或 `FLAC` 格式。提供 **GUI 图形界面** 和 **命令行** 两种使用方式。

## 🚀 快速开始

### 环境要求

- **操作系统**：Windows 10 / 11
- **Python**：3.8 或更高版本（需含 tkinter，Python 安装时默认自带）

### 方式一：双击启动（推荐）

直接双击 `start_ncm.bat` 即可打开 GUI 界面。

### 方式二：命令行启动

```bash
python ncm_converter.py
```

### 方式三：命令行批量转换

```bash
python ncm_cli.py -d "D:/Music/NCM" -r
```

## 🖥️ GUI 图形界面使用指南

### 1. 选择目录

打开程序后，在顶部配置区域设置：

- **NCM 源目录**：包含 `.ncm` 文件的文件夹路径
- **输出目录**：转换后的 MP3/FLAC 保存位置（留空则输出到源目录）

> 💡 **跨盘输出优化**：当源目录不在系统盘（C 盘）时，程序会自动将输出目录设为 `C:\Users\<用户名>\Music\NCM_Output`，以避免外接磁盘（如 USB SSD）的随机写入瓶颈，转换速度可提升 **100~1000 倍**。用户可手动修改输出目录覆盖此设置。

可以通过右侧 **"浏览…"** 按钮打开文件夹选择对话框。

### 2. 配置选项

| 选项 | 说明 |
|------|------|
| **递归子文件夹** | 勾选后扫描源目录下所有子文件夹中的 .ncm 文件 |
| **转换后删除源文件** | 转换成功后自动删除原始 .ncm 文件 |
| **跳过已转换文件** | 检测输出目录中已有同名 MP3/FLAC 文件则跳过 |
| **并行批次数** | 同时处理的文件数量（1-4），默认 3 |

### 3. 扫描文件

点击 **"🔍 扫描"** 按钮，程序会：

1. 扫描目录中所有 `.ncm` 文件
2. 在文件列表中显示文件名、大小、路径
3. 自动标记已转换过的文件（✅）和待转换的文件（⏳）
4. 底部状态栏显示扫描统计

### 4. 开始转换

点击 **"🚀 一键转换"** 按钮开始批量转换：

- 文件列表中每个文件独立显示转换状态：
  - ⏳ 等待中
  - 🔄 转换中
  - ✅ 已完成
  - ❌ 失败
- 顶部进度条实时更新
- 底部状态栏显示当前处理进度

> 当所有文件均已转换时，点击"一键转换"会立即显示跳过统计，无需等待。

### 5. 查看日志

转换完成后自动弹出 **转换日志** 窗口，包含：

- 每个文件的转换结果（成功/失败/跳过）
- 输出文件名、格式、大小
- 转换速度（文件/秒）和总耗时
- 可一键复制日志内容

### 6. 取消转换

在转换过程中，点击 **"取消"** 按钮可以中止转换。已成功转换的文件不会被回滚。

## ⌨️ 命令行使用指南

命令行版 `ncm_cli.py` 适合脚本自动化和批量处理。

### 基本语法

```bash
python ncm_cli.py [选项] [文件...]
```

### 参数说明

| 参数 | 缩写 | 说明 |
|------|------|------|
| `files` | — | 指定 NCM 文件路径（可传入多个） |
| `--directory` | `-d` | NCM 文件所在目录 |
| `--output` | `-o` | 输出目录（默认为源目录） |
| `--recursive` | `-r` | 递归扫描子文件夹（需配合 `-d`） |
| `--remove` | `-m` | 转换成功后删除源 .ncm 文件 |

### 使用示例

```bash
# 转换目录下所有 NCM 文件
python ncm_cli.py -d "D:/Music/NCM"

# 指定输出目录
python ncm_cli.py -d "D:/Music/NCM" -o "D:/Music/Output"

# 递归扫描子文件夹
python ncm_cli.py -d "D:/Music/NCM" -r

# 递归 + 删除源文件
python ncm_cli.py -d "D:/Music/NCM" -r -m

# 转换指定的单个或多个文件
python ncm_cli.py song1.ncm song2.ncm song3.ncm

# 组合使用：指定文件 + 目录
python ncm_cli.py song1.ncm -d "D:/Music/NCM" -o "D:/Output"
```

## ⚙️ 配置文件

GUI 版会自动生成 `ncm_converter_config.json` 配置文件，保存用户偏好：

```json
{
  "last_input_dir": "D:/Music/NCM",
  "last_output_dir": "D:/Music/Output",
  "recursive": true,
  "remove_source": false,
  "skip_existing": true,
  "ncmdump_path": "ncmdump_bin/ncmdump.exe",
  "workers": 3
}
```

| 字段 | 说明 |
|------|------|
| `last_input_dir` | 上次使用的源目录 |
| `last_output_dir` | 上次使用的输出目录 |
| `recursive` | 是否递归子文件夹 |
| `remove_source` | 是否转换后删除源文件 |
| `skip_existing` | 是否跳过已转换的文件 |
| `ncmdump_path` | ncmdump.exe 的路径 |
| `workers` | 并行工作线程数（1-4） |

## 🔧 技术架构

### v3.7 架构特点

```
┌─────────────────────────────────────────────┐
│               GUI (tkinter)                  │
│  ┌─────────┐  ┌──────────┐  ┌────────────┐ │
│  │ 配置面板 │  │ 文件列表  │  │ 进度/状态  │ │
│  └─────────┘  └──────────┘  └────────────┘ │
│         │            ↑              ↑        │
│         ▼            │              │        │
│  ┌──────────────────────────────────────┐   │
│  │    GUI 更新队列 (coalescing + 批量)   │   │
│  │  每 50ms 处理，每批 ≤30 项后让出循环  │   │
│  └──────────────────────────────────────┘   │
│         ↑                                    │
│  ┌──────────────────────────────────────┐   │
│  │      ConversionEngine (转换引擎)      │   │
│  │  ┌─────┐ ┌─────┐ ┌─────┐ ┌─────┐   │   │
│  │  │ W-1 │ │ W-2 │ │ W-3 │ │ W-4 │   │   │
│  │  └──┬──┘ └──┬──┘ └──┬──┘ └──┬──┘   │   │
│  │     │       │       │       │       │   │
│  │  ┌──▼──┐ ┌──▼──┐ ┌──▼──┐ ┌──▼──┐   │   │
│  │  │dump1│ │dump2│ │dump3│ │dump4│   │   │
│  │  └─────┘ └─────┘ └─────┘ └─────┘   │   │
│  └──────────────────────────────────────┘   │
│                                             │
│  W = Worker Thread    dump = ncmdump.exe    │
└─────────────────────────────────────────────┘
```

### 核心机制

| 机制 | 实现方式 | 作用 |
|------|----------|------|
| 并行转换 | `ThreadPoolExecutor` + `n` 个 Worker | 多文件同时解密 |
| 取消响应 | `threading.Event` + 每秒轮询 | 毫秒级取消响应 |
| GUI 不卡顿 | 更新队列 + coalescing + 批量刷新 + 每批让出事件循环 | 防止界面冻结 |
| 跨盘输出 | 自动检测非 C 盘源目录，重定向输出到 C 盘 | 避免 USB 盘随机写入瓶颈 |
| 生命周期管理 | `_convert_done_pending` 标志 + 弹窗关闭后重置状态 | 防止异步回调竞态 |
| 系统不影响 | `BELOW_NORMAL_PRIORITY_CLASS` | 降低子进程优先级 |
| 超时保护 | `proc.wait(timeout=1.0)` 循环 | 单文件 5 分钟上限 |
| 调试日志 | 写入 `ncm_debug.log` | 卡死时可追溯原因 |

### 输出格式说明

NCM 文件解密后的格式取决于原始编码：

- 如果原始文件是 **FLAC 无损**，解密后输出 `.flac`
- 如果原始文件是 **MP3 有损**，解密后输出 `.mp3`

> ⚠️ **无法强制转换格式**：NCM → MP3 或 NCM → FLAC 取决于网易云音乐下载时的音质选择。

## 📝 版本历史

### v3.7 (当前版本)

**Bug 修复：**
- 修复全部文件已转换后再点击"一键转换"导致卡死的问题
- 修复取消转换后再点击"一键转换"导致未响应的问题
- 核心修复：`convert_batch` 全部跳过时提前返回未设置 `self.log_lines`，导致 worker 线程访问 `engine.log_lines` 时抛出 `AttributeError`，`_convert_done` 永远不会被调用，GUI 永久卡在 `converting=True`

**改进：**
- 全部文件跳过时不弹出日志弹窗，直接重置状态允许再次转换
- `_convert_done` 增加幂等保护，防止旧转换的异步回调干扰新转换
- Worker 线程增加异常安全包装，确保异常时也能正确清理状态

### v3.6

- 自动检测非系统盘（C 盘）源目录，将输出重定向到 `C:\Users\<用户名>\Music\NCM_Output`
- 跨盘输出可避免 USB 外接盘的随机写入瓶颈，实测转换速度提升 100~1000 倍（8.6MB 文件：75 秒 → 0.55 秒）
- 用户可手动修改输出目录覆盖此设置

### v3.5

- 修复 GUI 未响应问题：限制每批 GUI 更新数量（≤30 项），处理后立即让出事件循环
- 合并重复的进度/状态更新（coalescing），大幅减少队列积压
- 完成后批量标记也分批处理，避免大量文件时界面冻结

### v3.4

- 修复 GUI 扫描后按钮消失的问题（调整 pack 布局顺序）
- 修复 GUI 卡死问题：使用队列批量处理 GUI 更新
- 使用 `threading.Event` 替代轮询，取消响应更快
- 修复跳过文件时返回值不正确的问题
- 速度计算仅统计实际转换的文件

### v3.3

- 子进程降低优先级 (`BELOW_NORMAL`)，避免 I/O 风暴导致资源管理器卡死
- 添加调试日志功能 (`ncm_debug.log`)

## ❓ 常见问题

### Q: 转换后文件在哪里？

默认输出到与源 NCM 文件相同的目录。如果指定了输出目录，则保存到输出目录，且保留原始的子文件夹结构。

### Q: 为什么转换后是 FLAC 而不是 MP3？

NCM 文件内部存储的是原始音频数据。如果你在网易云音乐中下载时选择的是"无损音质"，解密后就是 FLAC；选择"标准"或"较高"音质则解密为 MP3。

### Q: 转换速度很慢 / 资源管理器卡顿？

- 如果源文件在外接磁盘（USB 硬盘/SSD），程序会自动将输出重定向到 C 盘，速度可提升 100 倍以上
- 尝试降低"并行批次数"为 1 或 2
- 工具会自动降低 ncmdump 进程优先级，一般不会影响系统
- 如果文件特别大（>100MB），单个文件可能需要更长时间

### Q: 部分文件转换失败怎么办？

- 检查 NCM 文件是否完整（未损坏）
- 查看转换日志中的错误信息
- 尝试手动运行：`ncmdump.exe 你的文件.ncm`

### Q: 如何只转换新下载的文件？

默认开启"跳过已转换文件"选项。只要输出目录中已有同名的 MP3/FLAC 文件（且大小 > 0），就会自动跳过。

### Q: 支持哪些操作系统？

目前仅支持 **Windows**。核心解密引擎 ncmdump 支持跨平台，但本工具的 GUI 使用了 Windows 特定的 `CREATE_NO_WINDOW` 和 `BELOW_NORMAL_PRIORITY_CLASS` 参数。
